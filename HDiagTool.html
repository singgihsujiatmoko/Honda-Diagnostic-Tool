Wake Up ECU Honda

Panduan Lengkap Keihin & Shindengen

Apa itu Wake Up ECU?

Wake Up ECU adalah proses mengaktifkan atau "membangunkan" ECU (Electronic Control Unit) dari mode sleep/standby agar dapat berkomunikasi dengan alat diagnostic atau programmer untuk keperluan tuning, repair, atau cloning.

Mengapa Diperlukan?

ECU Honda memiliki fitur sleep mode untuk hemat daya. Tanpa wake up, ECU tidak bisa diakses untuk programming.

Kapan Digunakan?

Saat melakukan read/write ECU, immo off, cloning ECU, repair checksum, atau tuning engine.

Perhatian!

Kesalahan dalam wake up atau wiring dapat merusak ECU. Pastikan menggunakan tegangan dan prosedur yang benar.

Jenis ECU Honda

Keihin ECU

  • Digunakan pada Honda Mobil & Motor
  • Processor: Renesas SH7058/SH7059
  • Wake up via K-Line atau CAN
  • Contoh: Jazz, Brio, Vario, PCX

Shindengen ECU

  • Umumnya pada Honda Motor
  • Processor: Renesas RL78/F13
  • Wake up via K-Line khusus
  • Contoh: Beat FI, Scoopy FI, Revo

Wake Up ECU Keihin

ECU Keihin Honda menggunakan protokol komunikasi K-Line (ISO 9141) atau CAN Bus tergantung model kendaraan.

Pinout Wake Up Keihin (OBD2)

Honda Mobil (OBD2 Port)

16 +12V Battery
4 Ground (Chassis)
5 Ground (Signal)
7 K-Line (ISO 9141)
6 CAN High
14 CAN Low

Honda Motor (DLC 4 Pin)

1 +12V (Dari Baterai/Kunci ON)
2 Ground
3 K-Line (Data)
4 Service Check (Optional)

Untuk PCX, Vario 150, ADV: Gunakan protocol ISO 14230 (KWP2000)

Prosedur Wake Up Keihin

Via K-Line (Bench Mode):

  1. 1 Hubungkan +12V ke pin power ECU
  2. 2 Hubungkan Ground ke pin GND ECU
  3. 3 Hubungkan K-Line adapter ke pin K-Line
  4. 4 Kirim init byte: 0xFE pada 10400 baud
  5. 5 Tunggu response sync dari ECU

Via CAN Bus:

  1. 1 Pastikan CAN termination 120Ω terpasang
  2. 2 Set baudrate: 500 kbps
  3. 3 Kirim wake up frame ke ID: 0x7DF
  4. 4 Data: 02 01 00 (Tester Present)
  5. 5 ECU response pada ID: 0x7E8

Wake Up ECU Shindengen

ECU Shindengen banyak digunakan pada motor Honda entry-level. Menggunakan K-Line dengan protokol khusus Honda.

Pinout Wake Up Shindengen

ECU Connector (26 Pin)

A1 +12V Main Power
A2 +12V Ignition (Kunci ON)
A3 Ground Power
B4 K-Line (DLC)
B5 Ground Signal

DLC Service Connector

1 K-Line Data
2 Ground

Shindengen hanya memerlukan K-Line dan Ground untuk komunikasi. Tidak ada CAN Bus.

Prosedur Wake Up Shindengen

Metode Standard (On Vehicle):

  1. 1 Putar kunci kontak ke posisi ON
  2. 2 Hubungkan K-Line adapter ke DLC
  3. 3 Baudrate: 10400 bps
  4. 4 Kirim init: 0xFE 0x04 0xFF 0xFF
  5. 5 Response valid: 0x04 0xFE

Metode Bench (ECU Lepas):

  1. 1 Siapkan power supply 12V/2A
  2. 2 Hubungkan +12V ke pin A1 dan A2
  3. 3 Hubungkan GND ke pin A3 dan B5
  4. 4 K-Line adapter ke pin B4
  5. 5 Jalankan software dengan timing 200ms

Alat yang Diperlukan

Berikut adalah peralatan yang dibutuhkan untuk melakukan wake up dan programming ECU Honda.

Hardware

K-Line Adapter (ELM327/KKL)

Interface K-Line to USB. Pilih yang support ISO 9141-2 dan ISO 14230

OBD2 to DLC Cable

Kabel konversi dari OBD2 standar ke Honda DLC 3/4 pin untuk motor

Power Supply 12V

PSU stabil 12V minimal 2A untuk bench mode. Gunakan yang ada proteksi

Bench Adapter/Wiring Board

PCB khusus untuk koneksi ECU di meja kerja dengan pin header

CAN Bus Adapter (Optional)

Untuk Honda terbaru yang menggunakan CAN protocol. Contoh: PCAN, Kvaser

Software

HDS (Honda Diagnostic System)

Software resmi Honda untuk diagnostic. Versi clone tersedia

HondaECU / ECU Flasher

Open source tool untuk read/write ECU Honda. Support Keihin

WinOLS / ECM Titanium

Software tuning untuk edit map dan parameter ECU

Terminal/Serial Monitor

PuTTY, Realterm, atau HTerm untuk debug komunikasi K-Line

Checksum Calculator

Tool untuk repair/recalculate checksum setelah edit ECU file

Tips Keamanan & Wiring

  • Selalu backup file ECU original sebelum modifikasi
  • Gunakan power supply dengan proteksi short circuit
  • Periksa tegangan dengan multimeter sebelum koneksi
  • Jangan cabut kabel saat proses read/write berlangsung
  • Jangan gunakan tegangan lebih dari 14V
  • Jangan membalik polaritas (+/-)
  • Jangan write file dari ECU lain tanpa edit
  • Jangan interupsi proses flashing
